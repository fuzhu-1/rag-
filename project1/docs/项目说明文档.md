# Enterprise-RAG 项目说明文档

## 1. 项目概述

Enterprise-RAG 是一个企业级知识库问答系统，核心思路是 **RAG（Retrieval-Augmented Generation，检索增强生成）**：先检索相关文档，再让大模型基于文档内容回答问题，从而有效抑制大模型的"幻觉"问题。

### 核心流程（一句话）

> 用户提问 → 向量检索 + 关键词检索 → 重排序 → 取 Top-5 最相关片段 → 喂给大模型 → 生成带来源引用的答案

***

## 2. 关键概念解释

### 2.1 什么是 Embedding（向量嵌入）？

Embedding 是将文本转换成数学向量的技术。语义相近的文本，在向量空间中距离也更近。

```
"如何使用百炼的embedding模型?"  →  [0.12, -0.34, 0.56, ..., 0.78]  (1024维向量)
"怎样把文本变成向量?"          →  [0.11, -0.32, 0.55, ..., 0.79]  (高度相似 ↑)
"今天天气真不错"              →  [-0.45, 0.67, -0.23, ..., 0.12]  (差异很大 ↓)
```

本项目使用 **BAAI/bge-m3** 模型，一次编码同时输出：

- **Dense 向量**（1024 维浮点数）：语义相似度计算
- **Sparse 向量**（词权重）：关键词匹配（类似 BM25）

### 2.2 什么是 RAG？

| 对比   | 纯大模型          | RAG 系统      |
| ---- | ------------- | ----------- |
| 回答依据 | 训练记忆（可能过时/错误） | 实时检索的文档内容   |
| 幻觉问题 | 严重，容易编造       | 大幅减少，有来源可追溯 |
| 知识更新 | 需重新训练         | 上传新文档即可     |
| 可解释性 | 无从考证          | 每个答案附带文档来源  |

### 2.3 混合检索是什么？

单一检索有盲区：

- **纯向量检索**：对"年假政策"和"休假制度"能找到，但对精确关键词"8888"可能漏掉
- **纯关键词检索**：能精确匹配"IT支持电话"，但不懂"怎么联系技术部门"是同一个意思

所以采用 **混合检索 = Dense ANN × 0.7 + BM25 × 0.3**，互相补充。

### 2.4 为什么要 Reranker？

初步检索取 Top-20，但排序不够精准。BGE-Reranker-v2-m3 对每个（问题, 文档片段）对做精细相关性打分，重新排序后只取 Top-5，显著提升准确率。

## 4. 各模块详细说明

### 4.1 loader.py — 文档加载器

**功能**：将各种格式的文档解析为纯文本。

| 格式       | 解析库                  | 处理方式            |
| -------- | -------------------- | --------------- |
| PDF      | PyMuPDF (fitz)       | 提取文本，无文本时自动 OCR |
| DOCX     | python-docx          | 提取段落 + 表格       |
| XLSX/CSV | openpyxl / csv       | 转为 Markdown 表格  |
| PPTX     | python-pptx          | 逐页提取文本框和表格      |
| TXT/MD   | 内置 open              | 直接读取            |
| PNG/JPG  | pytesseract + Vision | OCR + 多模态图片描述   |

**安全机制**：

- 扩展名校验（白名单）
- 路径遍历防护（检测 `..`）
- 文件大小限制（默认 100MB）

**调用方式**：

```python
from src.loader import DocumentLoader

loader = DocumentLoader()
docs = loader.load_file("data/demo/员工手册_2024.md")
# 或批量加载目录
docs = loader.load_directory("data/uploads/")
```

### 4.2 splitter.py — 文本分割器

**功能**：将长文档切分成适合检索的小块。

**分割策略**：

1. 递归字符分割：按优先级 `\n\n → \n → 。→ ！→ ？→ 空格` 依次尝试
2. 语义分块（可选）：检测 Markdown 标题（`##`），按章节分割
3. 参数：`chunk_size=512`（每块最多 512 字符），`overlap=100`（相邻块重叠 100 字符）

**为什么要重叠？** 防止关键信息正好落在分割边界上被截断。

```python
from src.splitter import TextSplitter

splitter = TextSplitter(chunk_size=512, chunk_overlap=100)
chunks = splitter.split(documents)
# 返回: [Document(content="...", metadata={...}), ...]
```

### 4.3 embedder.py — 向量嵌入器

**功能**：将文本块转为向量，同时构建 BM25 关键词索引。

**BGE-M3 模型特点**：

- 支持中英多语言
- 单模型同时输出 Dense + Sparse
- Dense 向量 1024 维，支持余弦相似度
- Sparse 权重可用于关键词检索

```python
from src.embedder import Embedder

embedder = Embedder()

# 为文档生成向量
dense_vectors, metadata = embedder.embed_documents(chunks)

# 为查询生成向量
query_vec = embedder.embed_query("年假有多少天？")

# BM25 关键词搜索
bm25_results = embedder.search_bm25("年假 政策", top_k=10)
```

### 4.4 retriever.py — 检索器

**功能**：执行混合检索 + 重排序，返回最相关的文档片段。

**检索流程**：

```
用户问题
   │
   ├──→ Dense ANN 检索 (余弦相似度) ──→ Top-20
   │
   ├──→ BM25 关键词检索 ──────────→ Top-20
   │
   ▼
加权融合 (Dense 0.7 + BM25 0.3)
   │
   ▼
BGE-Reranker-v2-m3 重排序
   │
   ▼
返回 Top-5 最相关片段
```

```python
from src.retriever import Retriever

retriever = Retriever(embedder)
retriever.index(dense_vectors, metadata, texts)

results = retriever.retrieve("公司的年假政策是什么？", top_k=5)
# 返回: [{"text": "...", "metadata": {...}, "score": 0.92}, ...]
```

### 4.5 generator.py — 答案生成器

**功能**：基于检索到的上下文，调用 LLM 生成答案。

**Prompt 工程策略**：

1. **CoT 链式推理**：先让模型分析问题、逐一检查上下文、提取关键信息，再生成最终答案
2. **防幻觉机制**：System Prompt 明确"如果参考文档中没有相关信息，请明确回答无法回答"
3. **来源引用**：Prompt 中要求引用文档名和页码
4. **对话历史**：保留最近 3 轮对话，支持追问

```python
from src.generator import Generator

generator = Generator()

# CoT 推理模式
result = generator.generate_with_cot(
    query="年假有多少天？",
    contexts=retrieved_contexts,
    history=[{"role": "user", "content": "..."}, ...],
)
# result = {"answer": "...", "reasoning": "..."}

# Query 扩展（生成近义问题提高召回）
variations = generator.expand_query("年假政策")
# → ["员工休假规定是什么？", "公司年假有多少天？"]
```

### 4.6 pipeline.py — 总管道

**功能**：串联所有模块，提供统一的 API。

```python
from src.pipeline import get_pipeline

pipeline = get_pipeline()

# 1. 摄入文档
pipeline.ingest_directory("data/demo/")

# 2. 问答
result = pipeline.query("公司的年假政策是什么？", use_cot=True)
# result = {"answer": "...", "reasoning": "...", "contexts": [...]}

# 3. 文档管理
chunks = pipeline.get_chunks()
pipeline.delete_document("员工手册_2024.md")
```

## 7. 配置说明

### 7.1 切换 LLM 提供商

```yaml
# 本地 vLLM（默认）
llm:
  provider: "local"
  api_base: "http://localhost:8000/v1"

# 切换至 OpenAI
llm:
  provider: "openai"
  openai:
    api_key: "${OPENAI_API_KEY}"
    model_name: "gpt-4o-mini"
```

### 7.2 切换向量数据库

```yaml
# Milvus（生产级）
vector_db:
  backend: "milvus"

# Chroma（零配置，适合开发）
vector_db:
  backend: "chroma"

# Qdrant（高性能 Rust）
vector_db:
  backend: "qdrant"
```

### 7.3 调整检索参数

```yaml
retrieval:
  vector_weight: 0.7     # 向量检索权重（提高 = 更重语义）
  bm25_weight: 0.3       # 关键词权重（提高 = 更重精确匹配）
  candidate_top_k: 20    # 粗排候选数
  final_top_k: 5         # 最终返回数

splitter:
  chunk_size: 512        # 文本块大小
  chunk_overlap: 100     # 重叠量
```

## 10. 扩展开发指南

如需添加新的文档格式支持，在 `loader.py` 中：

1. 在 `ALLOWED_EXTENSIONS` 中添加扩展名
2. 实现 `_load_xxx(self, path)` 方法
3. 在 `load_file()` 的 `dispatch` 中添加分支
4. 在 `config.yaml` 的 `security.allowed_extensions` 中添加扩展名
